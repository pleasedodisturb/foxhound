# Application Packages

Each application gets its own directory: `[company]-[role-slug]/`

## Directory Structure

```
cv/applications/
├── example-company-ai-engineer/
│   ├── cover-letter.md          # Markdown source (you write this)
│   ├── cover-letter.pdf         # Generated by md_to_pdf_cover_letter.py
│   └── your-name-company-cv.pdf # Generated by render_tailored_cvs.py
├── another-company-tpm/
│   ├── cover-letter.md
│   ├── cover-letter.pdf
│   └── your-name-company-cv.pdf
└── README.md                    # This file
```

## Workflow

1. Create a directory: `mkdir cv/applications/company-role/`
2. Write `cover-letter.md` (see template below)
3. Add role to `tools/render_tailored_cvs.py` ROLES dict
4. Add directory to `tools/md_to_pdf_cover_letter.py` dirs list
5. Run both scripts to generate PDFs

## Cover Letter Template

```markdown
Your Name
your.email@example.com · +49 XXX XXXXXXX · City, Country

---

March 2026

**Re: [Role Title] at [Company Name]**

Opening paragraph — why this role, why this company. Be specific.

Middle paragraphs — your relevant experience. Back every claim with a concrete story.

- **Key achievement:** Quantified result from relevant experience
- **Technical depth:** Specific tools, systems, or architectures you built
- **Why you:** What makes you uniquely suited for this role

Closing — what excites you, availability, sign-off.

Your First Name
```

## Notes

- Cover letters are gitignored (they contain personal data)
- Only the directory structure pattern is committed
- PDFs are generated locally, not checked in
